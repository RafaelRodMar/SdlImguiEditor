Menu bar : file, edit, build, views, settings, tools, help.
Tools: hierarchy/project, level, layers, objects, log/console, inspector.
Resources: sprites, animation, mesh, sound, music, fonts.
Map editor. Add and manipulate sprites on the scene. Save in JSON format.
Atlas: texture + text file describing the sprites in the texture. (format: id + x,y,w,h)
Sprites preview and manipulation: add/remove from canvas, select sprite one by one, move sprites with mouse, rotate and scale.
Working directory and saving: one directory for each scene with all the related things there. JSON format allows easy loading.
Layers: add/delete, move up/down, show/hide, move entire layer horizontally/vertically. each layer is in a different JSON file, each layer is a vector of sprites.
Sprite manipulations: copy selected sprite to left, right, up, down, diagonal. Useful for tilemap scene. Flip horizontally/vertically, delete sprite, change color.
Canvas : change color, shows position of the mouse.
Log: describes what's happening and is saved with all the project files. engine_log("message")
Auto Save: every 30 seconds.
Mesh editor: objects made with connected points.
Object Node (Entity): everything in a scene will be encapsulated in a ObjectNode and attached to a parent.
Different objects:
primary objects: SpriteObject, LayerObject, MeshObject, PhysicObject, AnimationObject, LightObject, SoundObject, MusicObject, ParticleObject, ShaderObject.
Logical objecs: combining two primary objects. MeshedObject( MeshObject attached to SpriteObject), SolidObject (SpriteObject to a PhysicObject).
Layers: objects are stored in layers and each layer only contains that kind of object. The engine can create layers to separate different kind of objects.
Mesh, Mesh Object and Mesh Editor:
Engine modes and submodes :
- Scene edition mode: allows to build a scene. Submodes:
    - object: allows to handle any type of object. Actions: add, delete, move, rotate, scale, duplicate.
    - mesh: allows the edition of mesh objects in the scene.
    - play: allows to test the game in real time.
Mesh: there are four types: polygon, circle, line, chain. A line is made of two vertex, adding one more converts it into a chain.
MeshObject: encapsulates a mesh.
MeshEditor: mesh submode activates the meshEditor. User can : add/remove/move vertex, move edge, edge extrusion.
Solid Object: adding a sprite and a mesh you get a MeshedObject. The mesh can be dynamic, static or kinematic. In play mode the engine generates the physic object from the mesh.
Name of entities: to retrieve them. Object o = engine.findObject("name")
Categories of entities: for collision logic. can_collide(Player, Enemy)
Grid.
Gravity.
Undo and redo actions.


The Left : The left side of the Canvas : Engine Mode, Save, Load, Sprite Tab, Mesh Tab, Color Tab
The Center : The Canvas, the utility buttons on top the Canvas, the Log window bellow the Canvas
The Right : The right side of the Canvas : Layer, Mesh, Object, Scene, Draw Checkbuttons, etc.


Collisions: four methods: begin, end, presolve and postsolve. Begin and end are called when there is the first contact and when the contact ends. The other are called arternatively and repeatedly while the objects are overlapping.
Actions: ActionObject, register actions in objects: my_player.registerAction<JumpAction>("jump") --->  if(key j is pressed) m_player.callAction("jump").
Objects in the hierarchy: find, move, remove.
Collisions: cancollide("obj1", "ojb2")
Configuration files: JSON file with resources: name of the type of resource (font, sound, music, texture, etc), folder, extension, default.
Animations: the editor read the textures and the json files. It build automatically the animations.
Animations has a name (player) and one or more sequences (idle, walk, run, jump). The default sequence for player is idle, so idle can be substituted by player for easyness.
Sequences tab: its empty until you select an animation. Then it's filled with the sequences, and each one contains: loop or not, frame rate.
Animations can be static telling them to not update and changing their frame when necessary. A breakable object can show the frame 0 until it breaks, then you set it to use the frame 1.
Layers of the same type can be merged.
Text object: it can be moved, rotated, scaled, duplicated..., you can change its font, size, outline thickness, style(bold, italic, underline, linethrough), color. For newline can be used the | character.
Project manager: when the engine starts.
